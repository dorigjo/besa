The AI agent execution control plane.

Before an AI agent touches a tool, API, database, or pipeline — Besa checks whether the action is declared, policy-approved, within budget, and attributable. Fail-closed. Signed proof of every decision.

The problem

AI agents are calling CRMs, payment APIs, deployment pipelines, and databases. When something goes wrong — or when compliance asks — your team needs to answer: What was this agent authorized to do? Was the action inside policy? Can you prove it? Most teams today cannot.

The category

Approve. Enforce. Prove.

Besa gives every AI-agent tool call a policy gate and a signed audit record. Declare what your agents are allowed to do. Enforce that policy before each call. Issue a cryptographic receipt for every decision — allow or deny.

What Besa proves
  • What your agent was authorized to do — by signed declaration
  • Whether each tool call was inside declared policy
  • Whether the call was admitted or denied — and the machine-readable reason
  • The exact request that triggered the decision
  • Which key signed the evidence — fully verifiable by any party
CLI 
npm install @dorigjo/besa

npx besa --help

besa keys
besa sign examples/manifest.yaml
besa trust add examples/manifest.signed.json
besa verify examples/manifest.signed.json
besa admit examples/manifest.signed.json crm.lookup
besa admit examples/manifest.signed.json crm.delete
besa receipt crm.lookup examples/manifest.signed.json
besa verify-receipt .besa/receipts/receipt.json examples/manifest.signed.json
besa keys rotate

Status

Public beta (0.1.0-beta.5). Local-first. The execution control artifacts — signed manifests, policy gates, signed receipts — are production-quality cryptography. Hosted infrastructure (verifier API, receipt retention, HSM) is on the roadmap.
