# v0.1.0-beta.1 - explicit trust and key continuity

This beta closes the largest trust gap in the original local MVP: a valid
signature is no longer treated as trusted merely because its public key is
embedded in the artifact.

## Added

- Versioned JSON trust stores with `active`, `retired`, and `revoked` keys.
- Explicit consumer pinning through `besa trust add`.
- Old-key-signed rotation proofs through `besa keys rotate`.
- Rotation propagation through `besa trust apply`.
- Trust inspection and emergency revocation commands.
- Cross-process ActionMeter locking and atomic meter writes.

## Trust semantics

- `active`: valid for verification and new admission.
- `retired`: valid only for artifacts timestamped before retirement.
- `revoked`: invalid for current and historical verification.

The rotation proof is signed by the previous private key. A consumer that
already trusts that key can therefore adopt the next public key without a new
trust-on-first-use step.

## Operational note

After publisher key rotation, re-sign active manifests. Existing receipts and
manifests remain verifiable against the retired key when they predate the
rotation.

This remains a local developer beta. Private keys are unencrypted local files,
and the meter lock coordinates processes on one host only.
