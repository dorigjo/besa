# v0.1.0-beta.0 - local trust-chain beta

Besa's first beta hardens the complete local trust-artifact loop.

## Highlights

- Signed manifest inputs are runtime-validated before cryptographic checks.
- Malformed manifests and receipts fail closed with explicit reason codes.
- Receipt signatures are bound to the manifest signing key.
- `besa receipt` can hash a real JSON request with `--request`.
- `besa verify-receipt` verifies the manifest, manifest hash binding, key ID,
  and receipt signature as one trust chain.
- ActionMeter state is scoped by manifest hash and fails closed when corrupt.
- Smoke tests run in an isolated temporary workspace.
- CI covers Node.js 20, 22, and 24 plus build, tests, smoke, and package checks.

## Install after publication

```powershell
npm install @dorigjo/besa@beta
```

## Status

Beta. Local-first. No hosted backend. Not production-ready.
