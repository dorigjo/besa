# v0.1.0-beta.2 - hardened trust artifacts

This beta hardens the cryptographic and packaging boundaries found during an
adversarial release review of `beta.1`.

## Security changes

- Manifest signatures cover the complete envelope, including `signedAt`,
  `manifestHash`, algorithm, public key, and public key ID.
- DER key parsing explicitly requires Ed25519 keys.
- Canonicalization rejects undefined values, non-finite numbers, class
  instances, functions, symbols, and circular references.
- Signed manifests reject unknown unsigned top-level fields.
- Meter lock cleanup verifies ownership before removing a lock.
- CLI flags fail closed when unknown, duplicated, or missing a value.

## Release verification

The release suite now packs Besa, installs the tarball in an empty temporary
project, imports the SDK by package name, invokes the generated `besa` binary,
and executes sign, verify, admit, receipt, and receipt verification.

## Compatibility

The signed-manifest envelope changed intentionally. Manifests produced by
earlier prereleases must be re-signed with `0.1.0-beta.2`.

This remains a local developer beta. Artifact timestamps are signed but are not
backed by an external trusted timestamp authority.
